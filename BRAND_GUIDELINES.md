## üé® Logo & Visual Identity

### **Primary Logo**
The Lunotes logo consists of:
- **Icon**: Document/note symbol representing knowledge and organization
- **Wordmark**: "LUNOTES" in bold, modern typography
- **Tagline**: "by Lunetix" in smaller, lighter text

### **Logo Variations**

#### **1. Primary Logo (Horizontal)**
- **Usage**: Main applications, headers, business cards
- **Dimensions**: 800x300px (recommended)
- **Background**: Gradient yellow to orange

#### **2. Icon Logo (Square)**
- **Usage**: App icons, social media profiles, favicons
- **Dimensions**: 512x512px
- **Background**: Gradient yellow to orange with rounded corners

#### **3. Circular Logo**
- **Usage**: Profile pictures, badges, stamps
- **Dimensions**: 512x512px
- **Background**: Gradient yellow to orange, fully circular

### **Logo Construction**
- **Icon Size**: 25% of total logo width
- **Text Size**: 75% of total logo width
- **Spacing**: 0.5x icon width between icon and text
- **Minimum Size**: 120px width for horizontal, 64px for icon-only

### **Clear Space**
Maintain clear space around the logo equal to the height of the "L" in "LUNOTES" on all sides.

#### **Font Weights**
- **Regular (400)**: Body text, descriptions
- **Medium (500)**: Subheadings, labels

- **H1**: 48px / Bold / -2px letter-spacing
- **H2**: 36px / Semibold / -1px letter-spacing
- **H3**: 24px / Semibold / -0.5px letter-spacing
- **H4**: 20px / Medium / normal letter-spacing

#### **Body Text**
- **Large**: 18px / Regular / 1.6 line-height
- **Regular**: 16px / Regular / 1.5 line-height
- **Small**: 14px / Regular / 1.4 line-height
- **Caption**: 12px / Medium / 1.3 line-height

#### **Interactive Elements**
- **Buttons**: 16px / Semibold / uppercase
- **Links**: 16px / Medium / underline on hover
- **Labels**: 14px / Medium / normal case

### **Typography Guidelines**
- **Line Height**: 1.5x font size for body text
- **Paragraph Spacing**: 1em between paragraphs
- **Maximum Line Length**: 65-75 characters
- **Contrast**: Minimum 4.5:1 ratio for accessibility

## üó£Ô∏è Voice & Tone

### **Brand Voice**
**Friendly Professional** - Approachable yet competent, helpful without being condescending

### **Tone Characteristics**

#### **Helpful**
- Provide clear, actionable guidance
- Anticipate user needs and questions
- Offer solutions, not just descriptions

#### **Confident**
- Use definitive language when appropriate
- Avoid unnecessary qualifiers ("maybe", "perhaps")
- Present features as benefits

#### **Warm**
- Use inclusive, welcoming language
- Show empathy for user challenges
- Celebrate user achievements

#### **Clear**
- Use simple, direct language
- Avoid jargon and technical terms
- Structure information logically

### **Writing Guidelines**

#### **Do:**
- Use active voice: "Create beautiful notes" vs "Notes can be created"
- Write in second person: "Your notes" vs "User notes"
- Use contractions: "You'll love" vs "You will love"
- Keep sentences under 20 words
- Use bullet points for lists

#### **Don't:**
- Use technical jargon without explanation
- Write in passive voice unnecessarily
- Use corporate speak or buzzwords
- Make assumptions about user knowledge
- Use negative framing when positive is possible

### **Content Types**

#### **Interface Copy**
- **Buttons**: Action-oriented ("Create Note", "Save Changes")
- **Labels**: Clear and descriptive ("Category", "Search Notes")
- **Placeholders**: Helpful examples ("Enter note title...")
- **Error Messages**: Constructive and solution-focused

#### **Marketing Copy**
- **Headlines**: Benefit-focused and emotional
- **Descriptions**: Feature benefits, not just features
- **CTAs**: Clear value proposition ("Start Writing", "Download Free")

## ‚ùå Don'ts

### **Logo Don'ts**
‚ùå Don't stretch or distort the logo
‚ùå Don't use the logo smaller than minimum size
‚ùå Don't place logo on busy backgrounds without container
‚ùå Don't recreate or modify the logo elements
‚ùå Don't use outdated logo versions
‚ùå Don't rotate the logo at angles

### **Color Don'ts**
‚ùå Don't use colors outside the approved palette
‚ùå Don't use low contrast combinations
‚ùå Don't use yellow text on white backgrounds
‚ùå Don't use multiple bright colors together
‚ùå Don't use gradients except for approved brand elements

### **Typography Don'ts**
‚ùå Don't use fonts other than Inter for brand materials
‚ùå Don't use all caps for body text
‚ùå Don't use multiple font weights in the same text block
‚ùå Don't use decorative fonts for interface elements
‚ùå Don't use text smaller than 12px for body copy

### **Voice Don'ts**
‚ùå Don't use technical jargon without explanation
‚ùå Don't be overly casual or unprofessional
‚ùå Don't make promises the product can't keep
‚ùå Don't use negative or discouraging language
‚ùå Don't assume user expertise levels
